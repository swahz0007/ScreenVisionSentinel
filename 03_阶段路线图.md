# 阶段路线图

最后更新日期：2026-07-05

## 阶段 0：项目初始化

目标：建立工程、文档、最小程序和基础测试。

功能范围：

- 工程目录。
- 核心文档。
- PySide6 最小窗口。
- OCR、截图、检测、安全、动作、SQLite 接口。
- Mock OCR。
- pytest 和 ruff。

不做什么：

- 不做完整 OCR。
- 不做区域框选。
- 不做持续截图。
- 不做自动点击。
- 不做复杂规则引擎。

前置条件：

- 本地 Windows 项目目录可写。
- Python 3.11 或兼容版本可用。

开发任务：

- 初始化目录和配置。
- 编写最小代码骨架。
- 编写基础测试。
- 编写核心文档。
- 运行 ruff 和 pytest。

验收标准：

- 程序可启动或入口可导入。
- 启动不截图、不 OCR 循环、不执行动作。
- Mock OCR 测试通过。
- 安全控制默认阻止动作。
- 文档记录当前限制。

退出条件：

- 阶段 0 测试通过。
- `00_项目总控.md` 和 `06_开发日志.md` 已更新。

风险：

- 当前环境未安装 PySide6。
- GUI 无法在无显示环境中验证。

测试场景：

- 导入入口。
- 运行单元测试。
- 运行 ruff。
- 人工启动窗口。

## 阶段 1：只观察

目标：实现屏幕区域选择、截图、OCR 和变化记录，但不提醒、不动作。

功能范围：

- 鼠标框选监控区域。
- 保存区域坐标。
- 手动测试截图。
- 固定间隔截图。
- 本地 OCR 接入。
- 展示当前与上一轮识别结果。
- 判断内容变化。
- 记录日志。
- 必要时保存脱敏测试截图。

不做什么：

- 不做自动点击。
- 不做半自动动作。
- 不做临床判断。
- 不做高风险提醒策略。

前置条件：

- 阶段 0 通过。
- 明确截图区域配置格式。
- 选定至少一个真实 OCR 实验方向。

开发任务：

- 区域选择 UI。
- 区域配置读写。
- 单次截图按钮。
- 截图循环控制。
- OCR 引擎接入实验。
- OCR 结果显示。
- 变化检测记录。

验收标准：

- 用户可选择并保存区域。
- 用户可手动截图。
- 监控启动和停止可控。
- OCR 失败有明确提示。
- 日志不含敏感数据。

退出条件：

- 只观察流程稳定运行。
- 已记录多显示器、DPI、远程桌面验证状态。
- 在真实工作电脑上用离线包完成坐标拾取和批量读取。
- 目标字段能稳定出现在 `/ocr/batch` 的 `texts` 和 `summary` 中。

阶段判断：

- 如果明天夜班能在真实工作页面稳定获取所需参数，则“眼睛”阶段成立。
- 之后进入“声音 JSON 给 AHK/按键精灵消费”的自动化编排准备阶段。

风险：

- OCR 误识别。
- DPI 缩放坐标错位。
- 截图泄漏。
- 窗口移动导致区域不准。

测试场景：

- 单显示器截图。
- 多显示器截图。
- 125% 和 150% DPI。
- 远程桌面分辨率变化。

## 阶段 2：识别后提醒

目标：在识别结果满足规则时提醒用户。

功能范围：

- 关键词触发。
- 数字变化触发。
- 连续多次一致确认。
- 冷却时间。
- 声音提醒。
- Windows 通知。

不做什么：

- 不执行鼠标或键盘动作。
- 不自动确认临床结果。

前置条件：

- 阶段 1 稳定。
- OCR 误识别样例已记录。
- 提醒规则可配置。

开发任务：

- 规则配置。
- 触发条件评估。
- 冷却时间。
- 提醒通道。
- 提醒日志。

验收标准：

- 规则可复核。
- 提醒不重复轰炸。
- 误识别不会直接触发动作。

退出条件：

- 提醒功能稳定且可关闭。

风险：

- 误提醒。
- 漏提醒。
- 日志包含敏感文字。

测试场景：

- 关键词出现和消失。
- 数字变化。
- OCR 低置信度。
- 冷却时间。

## 阶段 3：半自动操作

目标：识别满足条件后弹出确认，用户确认后才执行动作。

功能范围：

- 动作预案。
- 用户确认弹窗。
- 执行前后截图或状态记录。
- 安全拒绝和紧急停止。

不做什么：

- 不做无人值守自动操作。
- 不做高风险或不可撤销操作。

前置条件：

- 阶段 2 稳定。
- 动作目标低风险、可撤销。
- 用户确认流程明确。

开发任务：

- 动作请求结构。
- 确认 UI。
- 动作执行器。
- 审计日志。

验收标准：

- 无确认不执行。
- 紧急停止有效。
- 执行前后有记录。

退出条件：

- 半自动流程可审计、可停止。

风险：

- 点击错位。
- 窗口移动。
- 重复执行。

测试场景：

- 用户拒绝动作。
- 用户确认动作。
- 紧急停止。
- 目标窗口移动。

## 阶段 4：有限全自动

目标：仅对低风险、可撤销、非临床决策类操作评估有限自动化。

功能范围：

- 自动动作总开关。
- 白名单动作。
- 冷却和次数限制。
- 审计日志。
- 回退方案。

不做什么：

- 不自动审核报告。
- 不自动发布报告。
- 不替代医生决策。
- 不自动修改关键数据。

前置条件：

- 阶段 3 稳定。
- 风险评估完成。
- 有明确停止和回退机制。

开发任务：

- 白名单配置。
- 多重确认策略。
- 自动动作审计。
- 失控保护。

验收标准：

- 默认关闭。
- 自动动作范围极窄。
- 可追溯、可停止、可恢复。

退出条件：

- 仅在低风险场景通过长期测试。

风险：

- 程序失控。
- 重复执行。
- 软件界面变化。

测试场景：

- 自动开关关闭。
- 白名单外动作拒绝。
- 超次数限制。
- 异常恢复。

## 阶段 5：稳定性与打包

目标：提升长期运行稳定性，并形成可分发包。

功能范围：

- 长时间运行测试。
- 多显示器适配。
- DPI 缩放适配。
- 异常恢复。
- PyInstaller 打包。
- 用户安装和升级说明。

不做什么：

- 不扩大自动化范围。
- 不绕过安全红线。

前置条件：

- 前序阶段功能稳定。
- 依赖和模型缓存策略明确。

开发任务：

- 长时间运行脚本。
- 性能和内存监控。
- 打包配置。
- 安装说明。
- 升级和回滚说明。

验收标准：

- 打包产物可启动。
- 配置和数据路径明确。
- 长时间运行无明显资源泄漏。

退出条件：

- 有可复现打包流程和验收记录。

风险：

- OCR 模型随包体积过大。
- Windows 权限问题。
- 杀毒误报。

测试场景：

- 全新 Windows 用户环境。
- 无管理员权限。
- 长时间运行。
- 打包后日志和配置路径。
