# 安全与风险控制

最后更新日期：2026-07-05

## 最高安全原则

- 先看清，再判断。
- 先记录，再提醒。
- 先确认，再操作。
- 不确定时不执行。
- 默认只读。
- 自动动作永久默认关闭。

## 视觉读取风险

| 风险 | 影响 | 当前策略 |
| --- | --- | --- |
| 任意区域读取 | 可能读取患者信息或隐私内容 | 仅本地调用；默认不保存调试截图；文档强调脱敏 |
| 截图范围过大 | CPU、内存压力或阻塞 | `CapturePolicy` 限制宽、高和总像素数 |
| 异常坐标 | 截图失败、资源浪费或不可预期行为 | 严格整数和坐标范围校验 |
| 调试截图泄漏 | 保存未脱敏界面 | 固定 `data/debug/`，默认关闭，调用者不能指定路径 |
| OCR 明文日志 | 泄漏患者姓名、检查号等 | Pipeline 和 Server 不记录 OCR 全文 |
| 本地 Server 暴露 | 被非预期程序调用 | 只绑定 `127.0.0.1`，不做远程服务 |
| Server 单请求阻塞 | 后续请求等待 | 当前 `HTTPServer` 串行；客户端应设置超时 |
| RapidOCR 并发不明 | 线程安全风险 | 不引入多线程 Server；单 Pipeline 实例串行调用 |
| OCR 误识别 | 错误读取界面状态 | 当前不把 OCR 结果绑定自动动作 |
| AHK 调试默认开启 | 自动保存截图或路径扩散 | `DEBUG_OCR := false`，只发送布尔调试开关 |

## 自动操作风险

| 风险 | 影响 | 当前策略 |
| --- | --- | --- |
| 点击错位 | 操作错误位置 | 当前不实现点击 |
| 自动提交 | 产生不可撤销结果 | 当前不实现提交 |
| 自动审核或发布 | 临床高风险 | 明确禁止 |
| 修改患者身份 | 数据安全和伦理风险 | 明确禁止 |
| 重复执行 | 多次触发动作 | 当前无动作；后续需冷却和确认 |
| 程序失控 | 后台持续执行动作 | 当前无后台动作；保留紧急停止接口 |

视觉读取风险和自动操作风险不得混为一谈。`CapturePolicy` 处理视觉输入边界；`SafetyController` 处理未来动作准入。

## 当前安全机制

- 默认只读观察模式。
- 自动动作总开关默认关闭。
- `SafetyController` 统一拦截动作。
- `DisabledActionExecutor` 当前拒绝所有动作。
- 紧急停止会关闭自动动作并进入停止状态。
- OCR 与动作执行解耦。
- CLI 和 Server 使用 `VisionPipeline`。
- Server 只绑定 `127.0.0.1`。
- Server 使用串行 `HTTPServer`。
- 调试截图目录加入 `.gitignore`。
- 截图、日志、数据库和本地配置默认忽略。

## 本轮已修复的视觉安全问题

- CLI `--save-debug <path>` 改为 `--save-debug` 纯开关。
- Server `save_debug` 改为只接受布尔值。
- 调试截图固定保存到 `data/debug/`。
- 调试截图文件名由程序生成，避免覆盖已有文件。
- Server 不再记录 OCR 全文。
- AHK 默认关闭调试截图。
- AHK 不再传递自定义调试截图路径。
- 建立 `CapturePolicy`，统一 CLI 和 Server 的区域边界。
- 建立 `VisionPipeline`，减少入口重复拼接截图/OCR逻辑。

## 当前实现状态

已实现：

- 自动动作默认关闭。
- 动作执行接口默认阻止。
- 紧急停止状态接口。
- Mock OCR 和 RapidOCR 回退。
- 鼠标拖拽框选监控区域，并本地保存。
- Qt 逻辑坐标到 mss 物理像素坐标换算。
- 用户手动触发的单次测试截图。
- CLI 和 Server 的统一视觉流水线。
- 调试截图固定目录和路径封闭。
- Server OCR 日志脱敏。

未实现：

- OCR 强制取消。
- 截图脱敏处理。
- OCR 置信度策略。
- 连续确认策略。
- 冷却时间。
- 动作白名单。
- 操作前后审计截图。
- 长时间运行资源监控。

## 本地 Server 边界

当前 Server 是本地辅助入口，不是远程服务：

- 只允许 `127.0.0.1`。
- 默认不随 GUI 自动启动。
- 无健康或 OCR 请求时不截图。
- 请求体大小限制为 8192 字节。
- 不记录请求体。
- 不记录 OCR 全文。
- 不提供远程认证或公网能力。

## 隐私控制

- 不提交真实患者姓名。
- 不提交住院号、身份证号、检查号。
- 不提交未脱敏医院软件截图。
- 测试图片使用虚构或脱敏内容。
- 默认本地 OCR。
- 云端 OCR 不作为默认方案。
- 本地日志、截图和数据库默认忽略。

## 医疗安全红线

项目不得自动审核报告、自动发布报告、自动删除患者数据、自动覆盖关键数据、自动修改患者身份信息、自动提交不可撤销结果，或替代医生进行诊断和临床决策。
