# AI 辅助开发规范

最后更新日期：2026-07-05

本项目中的“AI 辅助开发工具”包括 Codex 或其他可直接修改本地代码的 AI 开发工具。

## 工作规则

1. 每次开始工作前先阅读 `00_项目总控.md`。
2. 再阅读当前任务相关文档。
3. 不得脱离当前阶段开发后续功能。
4. 不得未经说明大规模重构。
5. 不得删除已有有效功能。
6. 修改前先检查现有实现。
7. 优先小步修改。
8. 每次修改后运行相关测试。
9. 测试失败不得宣称完成。
10. 不得伪造 OCR、Windows 或性能测试结果。
11. 无法验证的内容必须明确标注。
12. 不得默认开启自动点击。
13. 所有动作执行接口默认关闭。
14. 患者相关测试只使用脱敏或虚构数据。
15. 不得把真实患者数据提交到 Git。
16. 完成实质修改后更新：
    - `00_项目总控.md`
    - `06_开发日志.md`
    - 必要时更新 `07_问题与决策记录.md`
17. 每次任务完成后必须说明：
    - 修改了什么
    - 为什么修改
    - 测试了什么
    - 哪些尚未验证
    - 当前下一步任务
18. AI 不得建议将 OCR 结果直接绑定临床动作。
19. AI 不得因为旧文档未记录而删除合理的新入口，例如 CLI、Server 或 AHK。
20. AI 必须先确认代码真实状态，再依据审计报告修改。
21. AI 不得仅因实现类未显式继承 Protocol 就认定为安全问题。
22. AI 不得虚假声明 Server 并发模型。
23. AI 不得虚假声明 `future.result(timeout=...)` 能强制取消 OCR。
24. AI 完成实质修改后必须同步总控、开发日志和相关技术文档。

## 当前阶段边界

当前阶段是阶段 1：只观察。AI 开发工具只能推进区域选择、手动截图、Mock OCR 显示、结果展示和日志记录等只读观察功能。

不得在当前阶段擅自实现：

- 自动点击。
- 复杂规则引擎。
- 临床相关自动操作。
- 未经单独验收的长时间后台监控任务。

## 多入口与视觉核心规则

- GUI、CLI、Server、AHK 都是入口，不是项目主干。
- CLI 和 Server 已使用 `VisionPipeline`，后续修改不得重新分叉公共逻辑。
- GUI 尚未迁移到 `VisionPipeline`，迁移时不得破坏现有区域框选和 DPI 换算。
- Server 当前是标准库 `HTTPServer` 串行处理，不得误写成多线程。
- RapidOCR 线程安全尚未验证，不得为了并发而并发。
- 当前没有 OCR 强制取消机制，不得加入无效超时包装伪装成安全能力。

## 视觉安全规则

- `save_debug` 只能是布尔开关。
- CLI 不得接受调试截图路径。
- Server 不得接受调试截图路径。
- 调试截图只能保存到固定目录 `data/debug/`。
- 普通日志不得记录 OCR 全文、患者身份信息或调试截图路径。
- 区域限制由 `CapturePolicy` 负责，不应塞入 `SafetyController`。
- 动作准入由 `SafetyController` 负责，不应塞入 Vision Core。

## 文件管理规则

- 优先编辑现有文件。
- 不创建 `_new`、`_updated`、`_final`、`_v2`、`_copy` 等版本文件。
- 如需归档，使用已有归档目录。
- 不覆盖原始数据。
- 不提交日志、截图、数据库和本地配置覆盖文件。

## 完成汇报规则

任务结束时必须说明：

- 修改了哪些文件。
- 创建了哪些文件。
- 是否归档文件。
- 运行了哪些测试。
- 测试结果如何。
- 哪些内容尚未验证。
- 下一次最推荐做什么。
