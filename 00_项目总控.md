# 屏幕视觉哨兵项目总控

最后更新日期：2026-07-10

## 项目真实定位

ScreenVision Sentinel（屏幕视觉哨兵）是一个 Windows 本地通用屏幕视觉感知工具。它通过截取用户或本地程序指定的屏幕区域，并使用本地 OCR 读取其中的文字、数字和界面状态。

当前核心思想：

> 指哪里，看哪里，读取哪里。

**架构分工原则**：Python 服务端作为绝对中立的“眼睛”，只输出 JSON 数据；由外部的 AHK/按键精灵等脚本充当“大脑和手”，负责具体的医学业务规则裁判（如年龄与 QTC 标准校验）以及键盘鼠标自动化。

GUI、CLI、Server、AHK 都只是入口。项目主干必须是统一的 Vision Core。

## 当前实际架构

- GUI：PySide6 窗口，支持拖拽框选、手动输入、单次截图和测试 OCR；尚未迁移到 `VisionPipeline`。
- CLI：`src/screenvision_sentinel/cli.py`，已使用 `VisionPipeline`。
- Server：`src/screenvision_sentinel/server.py`，使用标准库 `HTTPServer` 串行处理请求，只绑定 `127.0.0.1`，已使用 `VisionPipeline`；提供 `/ocr`、`/ocr/batch`、单次 `/monitor/tick`、显式启动/停止的 `/monitor/start`、`/monitor/stop` 与不含 OCR 原文的 `/monitor/status`。
- AHK：`scripts/ahk_example.ahk` 是自包含按钮工作台，通过本地 Server 调用 OCR，默认关闭调试截图，支持自动选择目标窗口后拾取坐标，并明确显示当前坐标配置名称。
- Vision Core：`src/screenvision_sentinel/vision/`，包含 `VisionPipeline`、`CapturePolicy`、`VisionResult` 和 `DebugImageStorage`。
- 截图：`MssCaptureService`。
- OCR：默认 RapidOCR；初始化失败或未知引擎时回退 Mock OCR。
- 安全：`SafetyController` 和 `DisabledActionExecutor` 仍保持自动动作默认阻止。

- AHK 坐标更新从 `InputBox` 改为可滚动列表选择，避免后续字段在弹窗中不可见。
- AHK 批量读取使用 `/ocr/batch` 一次请求并汇总，未配置字段显示 `[未配置坐标]`，空读字段显示 `[空]`。
- AHK 单项和批量读取默认使用稳定模式 `fast_mode=false`；可由用户勾选“极速模式”优先速度。坐标写回后的即时 OCR 测试使用当前所选模式，并同步更新当前会话变量，不再自动重载。
- AHK 目标窗口选择统一归一到顶层窗口，降低 Gink 子窗口句柄导致的坐标偏移风险。
- AHK 主工作台显式使用 `Gui, 1:*` 显示和隐藏，避免参数选择子窗口改变默认 GUI 后主界面无法恢复。
- Server OCR 空读时最多进行一次相反识别模式的兜底，每个字段最多两轮，避免“读取全部”因空读重试而长时间阻塞；JSON 标记 `fallback_used` 与 `fallback_strategy`。
- Server `/ocr/batch` 对真实 `VisionPipeline` 使用共享截图优化，一次请求只抓一张大图，再在内存中切分字段区域 OCR；响应返回 `texts`、`elapsed_ms`、`shared_capture` 及不含识别原文的模式、回退、空读和耗时诊断字段。
- Server `/ocr/batch` 返回 `summary` 作为 AHK 显示汇总；如果共享截图结果全空，会自动回退到逐项稳定读取并标记 `shared_capture_fallback`。
- Server 支持可选字段结构类型 `text`、`number`、`date`、`datetime`、`gender`；返回 `validation_status`、`is_valid`、`requires_review` 与批量 `review_count`。校验只做空值、OCR 置信度和文本结构检查，不修正 OCR 原文、不做医学参考范围或诊断判断。
- 建立 `ObservationStabilityTracker`：单次和后台观察均对有效字段执行连续确认；`/monitor/status` 仅给出稳定/变化/计数元数据，不回传 OCR 原文、值指纹或坐标。
- 建立显式后台只读监测：`/monitor/start` 接收一次已验证的字段坐标并在内存中按 `0.5`–`60` 秒间隔轮询；`/monitor/stop` 在当前 OCR 轮结束后停止。服务启动不会自动监测，后台轮询禁止周期调试截图，并与普通 OCR 共用执行锁，避免并发使用 RapidOCR。
- 修复旧服务冲突：健康检查增加 `api_revision` 和 `background_monitor_available`；Server 独占绑定 `127.0.0.1:8181`，不再允许多份旧/新 Python 服务抢占同一端口导致后台接口随机 `404`。
- AHK 工作台已向单项和批量请求发送字段类型，遇到空读、低置信度或格式不符时显示“建议人工复核”；离线工作包显示“工作电脑（2026-07-08 人工测量）”，避免与本机调试坐标混淆。

- AHK 脚本统一保存为 UTF-8 with BOM，避免 AutoHotkey v1 中文乱码导致解析异常。

## 安全红线

- 默认只读。
- 所有自动操作默认关闭。
- 不确定时不执行。
- 不得自动审核、发布或提交临床结果。
- 不得自动修改患者身份信息。
- 不得自动删除或覆盖关键数据。
- 不得替代医生诊断或临床决策。
- 不得提交真实患者信息或未脱敏截图。
- OCR 结果不得直接绑定高风险动作。
- 调试截图不得由调用者指定保存路径。
- 普通日志不得记录 OCR 全文或调试截图路径。
